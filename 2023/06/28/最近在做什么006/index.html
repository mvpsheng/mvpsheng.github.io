在做什么

总结一下就是最近在高频率的做项目。
项目驱动学习。 有什么想法马上就去试试看。大概一两天做一个东西出来，目前更多的还是在做一些学习社区的 Challenges, 像 Frontend Mentor， devChallenges.io 都很不错，也会得到一些 feedback，项目也很精美，容易获得成就感和看到自己的成长。

  1. 技术栈 Frontend Mentor 的项目我主要用 HTML,CSS 和 JavaScript 来做。
  2. 实现自己 Idea 一般会用 React 来做（React 虽然目前用到的技术点比较少，但是比较顺手了）。

why

  • 前端感觉更容易获得一些学习资源或找到社区

  • 前端容易快速检测想法

    感觉很少有后端项目的 IDEA，最近学后端知识也比较少了。 但是想做个啥，前端很快就能实现，所以很多时间都放在前端学习了，也会经常回头再学一些 CSS 的特性。

感想与收获

  • 充分总结做过的项目，对成长有很大帮助。

    Frontend mentor 的社区挺活跃的，我发现有一些人只做过几个项目，但是每一个都很精美， 但是有一些人做了不少，但是做的很粗糙。 我发现粗糙的人 它的做完项目好像也不太会总结自己学到了什么，遇到的难点。 但是做得很精美的项目的人，通常他们的项目 README.md 文件都会有很详细的记录，总结，DEMO 地址，等等。 所以我觉得做一个项目，要充分的做好记录，比如学到了什么新东西，难点在哪里，记录下来，应该就是不断做项目能提取到的最大价值。也是程序员快速成长关键。

  • 节约实践，减少重复性工作

接下来做项目，有想法就马上动手试试，这个习惯是不错的，但是也要充分的构思一下，其中涉及到的数据啊，设计模式啊，技术栈啊。因为近期做项目感受就是项目初期构建的时候有很多重复的工作，我希望可以不用把时间花费再重复性的事情上。项目的价值，能更深入的理解什么技术，是个新颖的创意，能对生活有什么帮助。
